(Versión en español al final)

I am  happy to announce that the release of 0.4.0 version of KLog is
available.

KLog has now a email list(Klog-users at lists.berlios.de) for discussion 
about KLog, wishlist, bug report, ideas, suggestions tips on using, ...
The main language of the list if English.
Subscribe at:
		https://lists.berlios.de/mailman/listinfo/klog-users

From the Changelog:

10-Sep-2006 - 0.4.0

- New feature: It is possible to manage user-created local awards (TPEA, 
DOK, WAS, ...) KLog reads user defined awards in a special format.
- New feature: KLog reads ".adif" files also.(TNX DL5PD)
- New feature: It is possible to add/delete new DXServer clusters in the 
setup box.(TNX DL5PD)
- GUI Improved: IOTA continent is now "automagically" detected.
- Bug fixed: Not really a bug but now, the log is cleaner as it does not 
save the "STATE" for ALL the QSO, only does it if needed.
- Bug fixed: After modifying a QSO the band and mode did not return to 
the last used (TNX DL5PD).
- Bug fixed: If the band changed, without any call in the Call box, KLog 
showed "new one, work it"
- Bug fixed: If the band changed, when modifying KLog fooled the entity 
color band boxes.
- Bug fixed: KLog recognises /MM as maritime mobile station
- Bug fixed: When opening a new log, the number of QSO did not start 
from zero.
- New translation: KLog is now also in German.(TNX DL5PD)

KLog is in English, Spanish, Portuguese and German.

KLog can still have many bugs and other not yet implemented features.
You can download KLog from:

http://jaime.robles.nu/klog
and quickly from:
http://klog.berlios.de
http://developer.berlios.de/projects/klog/

You will find the tar.gz sources and the debian package (debian package 
only in my homepage)
You can see some screenshots in the web.

The KLog's new-releases email list is created using the addresses of 
people I have previously received emails from about KLog. Feel free to 
ask me to remove your address if you don't want to be alerted of new 
KLog releases.

Thanks for your help and I hope you find KLog useful.
73

Me alegro de anunciaros que ya está disponible la versión 0.4.0 de
 KLog.

Desde ahora KLog tiene una lista de correo(Klog-users at lists.berlios.de) 
para charlar sobre KLog, pedir funcionalidades, avisar de errores, 
ideas, sugerencias, trucos sobre cómo usarlo, ...
El lenguaje principal de la lista será inglés.
Subscribete en:
		https://lists.berlios.de/mailman/listinfo/klog-users


Del fichero Changelog vemos los cambios:

10-Sep-2006 - 0.4.0

- Nueva funcionalidad: Es posible gestionar diplomas creados por los 
propios usuarios It is possible to manage user-created local awards 
(TPEA, DOK, WAS, ...) KLog lee diplomas definidos por el usuario en 
unformato especial.
- Nueva funcionalidad:  KLog lee tambien ficheros ".adif" (no 
sólo .adi).(TNX DL5PD)
- Nueva funcionalidad:  It Es posible añadir servidores de DXCluster en 
la configuración.(TNX DL5PD)
- GUI mejorada: Se detecta el continente IOTA "automágicamente".
- Error solucionado: No era realmente un error pero ahora no guarda el 
campo STATE para todos los QSO, sólo para los que lo necesiten.
- Error solucionado: Después de modificar un QSO, la banda no volvía a 
la última utilizada (TNX DL5PD).
- Error solucionado: Si la banda cambiaba, sin tener un indicativo en el 
cuadro, KLog mostraba "Nuevo, trabájalo".
- Error solucionado: Si la banda cambiaba, al modificar, KLog confundía 
los cuadros de color de banda.
- Error solucionado: KLog reconoce /MM como estación movil marítima.
- Error solucionado: Cuando se abría un log, el número de QSO no se 
ponía a cero.
- Nueva traducción: KLog is now also in German.(TNX DL5PD)


KLog está disponible en inglés, español, portugués y alemán.

KLog puede tener aún muchos errores y otras funciones por implementar.

http://developer.berlios.de/projects/klog/

Debian sólo en mi página)
Puedes ver capturas de pantalla en la web.

dirección de la lista.
