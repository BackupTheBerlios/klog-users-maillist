 (Versión en español al final)

 I am  happy to announce that the release of 0.4.4 version of KLog is
 available.

The mailing list of KLog is in Klog-users at lists.berlios.de for 
discussion about KLog, wishlist, bug report, ideas, suggestions tips on 
using, ...

 The main language of the list if English.

 Subscribe at:

                 https://lists.berlios.de/mailman/listinfo/klog-users

 From the Changelog:

1-may-2008 - 0.4.4
- BUG fixed: KLog now recognises a "/LH" LightHouse stations (TNX G3OAG)
- BUG fixed: TLF import did not show the imported QSO in the log. QSO 
were not lost, just not shown. They were added to the logfile and saved 
when saving. No data was lost. This bug was introduced in release 0.4.2 
after an optimization in the logfile reading.
BUG: Frequency was not compliance with ADIF as was saved in KHz instead  
of MHz.
- BUG fixed: If a logfile does not provide a MY_GRIDSQUARE, KLog does 
not add the predefined MY_GRIDSQUARE.
- BUG fixed: Some GRIDSQUARES were not detected as wrong althought they 
were.
- Minor fix: Corrected the URL to CTY.DAT file in the start message.
- GUI improved: Added a progress dialog when saving.
- GUI changes: Some widgets have been changed to prepare for the Qt4 
migration. Specially the Led near the QRZ box that will reapear as usual 
when KLog is migrated to Qt4.
- Added the 70Mhz band.
- New feature: New tool added "Merge QSO data" that looks in all the log 
for the QSO that has been worked more than once and merges the data 
like Name, QSL info, QTH, Locator, ...
The information comming from all the appearances of a call in the log is 
grouped in the first appearance of that call in the log.

- Updated Swedish translation. (TNX SM4BE)
- Updated Spanish translation.  (EA4TV).

This release of KLog isin English, Spanish, Portuguese, Brazilian 
portuguese, German, Serbian, Galizian and Swedish.

 KLog can still have many bugs and other not yet implemented features.
 You can download KLog from the new KLog (and my own) page.

 http://jaime.robles.es/klog

 You will find the tar.gz sources and the debian package (debian
 package only after some days)
 You can see some screenshots in the web.

 The KLog's new-releases email list is created using the addresses of
 people I have previously received emails from about KLog. Feel free
 to ask me to remove your address if you don't want to be alerted of
 new KLog releases.

 Thanks for your help and I hope you find KLog useful.
 73

 Del fichero Changelog vemos los cambios:

1-mayo-2008 - 0.4.4

- Error corregido:  KLog reconoce los indicativos "/LH" LightHouse, de 
faros (TNX G3OAG)
- Error corregido:  La importación desde TLF no mostraba los QSO 
importados en el log. Los QSO no se perdían, pero no aparecían tras 
importar. No había pérdida de datos. Este error se introdujo en la 
versión 0.4.2 tras una optimización de la lectura del log.
- Error corregido:  La frecuencia no era compatible con las 
especificaciones de ADIF porque se guardaba en KHz en vez de MHz.
- Error corregido:  Si un log no proporciona el campo MY_GRIDSQUARE, 
KLog no añade el MY_GRIDSQUARE predeterminado.
- Error corregido:  Algunos locators no se detectaban como incorrectos 
aunque lo fueran.
- Error (menor) corregido:  Actualizada la URL al fichero CTY.DAT en el 
mensaje inicial.
- Interfaz gráfica mejorada: Añadido un cuadro de progreso al guardar.
- Interfaz gráfica mejorada: Se han cambiado algunos widgets de cara a 
la futura y próxima migración a Qt4. Especialmente el led cercano al al 
cuadro de QRZ que reaparecerá como siempre cuando KLog haya migrado a 
Qt4..
- Nueva funcionalidad:Added the 70Mhz band.
- Nueva funcionalidad: New tool added "Consolidar QSOs" que busca en 
todo el log los QSO que se han trabajado más de una vez y consolida los 
datos como nombre, información de QSL, QTH, locator, ... La información 
de todos los QSO se almacena en el primer QSO con ese indicativo.


- Actualizada la versión en sueco. (TNX SM4BE)
- Actualizada la versión en español. (EA4TV).
